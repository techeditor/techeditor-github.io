业务介绍

背景

生成式AI (Generative AI) 是一种人工智能技术，可以追溯到深度学习和神经网络。它通过学习大量数据并模拟人类的创造力，可以生成文本、图像、音频等内容，在多个领域展现出巨大的潜力，如自然语言处理、计算机视觉、音乐创作等。随着硬件和算法的不断进步，生成式AI正快速发展，为业务创新提供了新的可能性。

生成式AI对算力有较高的要求。由于生成式AI需要处理庞大的数据集和复杂的模型，其训练和推理的过程需要大量的计算资源。运行较大规模的生成式AI模型，通常需要高性能的硬件设备，如图形处理单元 (GPU) 或专用的AI加速器。这些设备能够并行处理大规模的矩阵运算和神经网络计算，以加快模型的训练和生成速度。此外，生成式AI的训练过程通常需要较长时间，甚至需要多天或数周才能完成。

随着生成式AI的发展，对算力的需求也在不断增加。更复杂的模型、更大规模的数据集和更高质量的生成结果，这些都需要更强大的计算能力来支持。因此，利用拥有弹性和可扩展的算力支持的云计算成为开展生成式AI实践的重要途径。通过云计算平台，生成式AI可以利用弹性的资源分配和并行计算能力，加快训练和推理速度，为创新和应用提供更广阔的可能性。

适用客户

适合对AI文生图有业务需求的企业客户，具体使用人员包括算法工程师、制作人、画师、设计师等。

适用场景

该方案为图像创作者和图像处理人员提供一种专业、安全、便捷的方式来调用云上资源，以帮助他们高效地完成文生图和图生图的工作。

客户痛点

在诸如游戏、建筑家装、出版、媒体等与图形图像设计相关的行业中，进行图像创作和处理的专业人士，在应用生成式AI技术时经常面临多种问题。例如：

  • 直接使用生成式AI算法，对编程能力要求很高，且工作量大。
  • 虽然开源的Stable Diffusion web UI有图形界面，降低了使用门槛，但是该项目是单机版，对安装设备的要求很高，需要配有GPU。
  • 因为单机版的计算资源限制，每次只能执行一项工作负载，这极大地影响工作效率。

客户价值

该解决方案对于客户业务的价值主要体现在以下方面。

加快客户AI生图业务的前期调研和模型验证的速度

该方案以开箱即用的插件形式，简化了客户在云上构建AI生图产品的过程，有助于客户缩短用于前期调研和模型验证的时间。

提升客户搭建自有AI生图平台和工具的效率

该方案通过提供API直接调用的方式，赋能客户加速构建自己的AI生图平台。

降低客户开展AI生图业务的硬件要求

该方案采用前端与后端分离的方式，将AI生图产品转变为瘦客户端，使客户在30分钟内在低配机型 (如CPU机型) 上即可构建轻量级AI生图应用，并集成Dreambooth、ControlNet、LoRA等常用功能组件。

助力客户规范地管理同生成式AI业务相关的资产

用户在使用Stable Diffusion web UI时默认以单机模式运行，训练的数据与模型、生成的图片等数字资产都存储在本地，企业难以统一管理。在采用该方案后，生成的资产都首先被存储在Amazon S3中，以便后续统一进行权限管理，例如只有管理员才可以下载，从而保证企业能够标准、可控地管理所有和生成式AI业务相关的资产。

技术介绍

概要

该解决方案帮助客户将基于Stable Diffusion web UI (简称web UI) 的模型训练、推理和调优等任务负载从本地服务器迁移至Amazon SageMaker，利用云上弹性资源加速模型迭代，避免单机部署所带来的性能瓶颈。该方案包括为社区提供的插件和Amazon CloudFormation模版两部分，目前支持为以下web UI原生/第三方插件提供云上工作流：txt2img、Img2img、LoRA、ControlNet、Dreambooth (含LoRA模型训练)、Image browser。该方案提供UI和API调用两种使用方式，满足客户多样化的使用场景需求。

特点

该解决方案主要具有以下特点：

安装便捷，开箱即用

使用Amazon CloudFormation一键部署亚马逊云科技的中间件，并以社区原生Stable Diffusion web UI的插件形式一键安装前端插件，客户即可快速利用Amazon SageMaker资源进行推理、训练和调优。

社区原生，与时俱进

该方案以Stable Diffusion web UI的插件形式运行，用户不需要改变现有的web UI使用习惯。方案插件和中间件的代码开源，采用非侵入式设计，有助于用户快速跟上社区相关功能的迭代，包括备受欢迎的Dreambooth、ControlNet和LoRA等插件。

扩展性强，迁移平顺

web UI界面与后端分离，web UI可以在任何支持的终端上启动而没有GPU的限制，原有的训练、推理等任务通过插件所提供的功能便捷地迁移到Amazon SageMaker上，从而为用户提供弹性计算资源、降低成本、提高灵活性和可扩展性。

部署前提条件

为确保成功部署和运行该方案，客户需要确保完成以下工作。

  • 客户需要拥有亚马逊云科技的全球区域账号。
  • 客户账号需要有足够的权限通过Amazon CloudFormation部署方案，所需最小权限请参考YAML文件。
  • 客户账号的Amazon SageMaker服务有足够的配额，包括1个ml.r5.large类型的endpoint来进行部署，至少1个ml.g4x或ml.g5x类型的endpoint来实现后续的推理和训练负载。

客户资源/能力评估

客户有涉及生图功能的业务，例如设计、营销等，并且近期对AI生图有快速验证 (POC) 或全面部署的需求。同时，要求客户了解Stable Diffusion web UI的基本使用方法。如果客户需要运维该解决方案，则应具有亚马逊云控制台的基本操作能力。如果客户需要构建AI生图产品，建议部署弹性Amazon SageMaker推理节点，以支持大量并发场景。

成本因素

该解决方案本身是开源免费的，主要使用成本为后端调用亚马逊云服务产生的费用。在部署该方案后，主要使用成本来自推理时的Amazon SageMaker Endpoint推理节点，和模型训练时的Amazon SageMaker Training用量。截至2023年6月，按照用户每天生成1000张图片，Dreambooth训练10次作为计算标准，在美国西部 (俄勒冈) 区域 (us-west-2) 使用该方案的成本约为每月583美元。

交付资源

该解决方案提供源代码和Amazon CloudFormation模版。客户可以参考实施指南，克隆GitHub上的资源后，执行脚本部署。客户也可以自行修改源代码来满足特定的需求。如需要咨询技术问题，请在GitHub上提交issue，或通过客户经理联系相关的技术支持人员。

伊克罗德 (eCloudrover)

伊克罗德信息推出了开箱即用的生成式AI解决方案——imAgine绘图解决方案，并在AWS Marketplace上架。imAgine以EC2 AMI的形式提供Stable Diffusion web UI应用，不仅集成了常用的插件与模型，还预安装了“Stable Diffusion亚马逊云科技插件”，可帮助用户将现有的工作负载 (推理、训练、ckpt合并等)从EC2迁移到亚马逊云环境。

使用该插件，用户可以将模型存储、推理任务、训练任务等组件完全和web UI剥离，使web UI回归控制台的单一属性。用户还可以在web UI上管理亚马逊云科技后端的计算资源，发起任务，并随时检查任务进度和结果，回溯任何一次推理的参数和输出。

更新说明

v1.5.0

新版本主要包括以下创新和功能优化：

  • 升级支持Stable Diffusion web UI v1.8.0
  • 支持换脸插件ReActor for Stable Diffusion
  • 新增API推理调试器功能
  • 支持基于Kohya_ss的LoRa模型训练
  • 优化云上资源管理的用户体验
  • 优化BYOC自定义插件的方式，并支持启动优化
  • 支持实时推理节点的弹性伸缩功能
  • 支持所有实例自定义实例数范围，即便自动伸缩，也可以设置最小实例数，而不是0
  • 优化中间件部署时间缩短67%，降低至4分钟
  • 升级Python版本从3.9至3.10
  • web UI资源管理界面用户体验优化

v1.4.0

新版本主要包括以下创新和功能优化：

  • 将Amazon SageMaker推理节点的冷启动时间缩短70%，减少到3分钟。
  • 支持通过Amazon SageMaker实时推理节点进行文生图推理。
  • 支持Stable Diffusion web UI v1.7.0。
  • 支持同一账号内在多region部署多套本解决方案。
  • 支持在云上通过ComfyUI进行推理。
  • 优化web UI用户体验，新增用户组、角色组、模型和推理节点列表的多维度过滤筛选，提供便利的增删功能。
