中国白，云体验—— 多智能体德化白瓷科普
White from China, Experience on Cloud

走向世界的中国白
Blanc de Chine Going Global

德化白瓷源于福建省德化县，拥有悠久的历史和独特的工艺特色，是中国国家地理标志产品。“中国白”(Blanc de Chine) 是法国人对明代德化白瓷的赞誉。

德化瓷器始于宋代，距今已有1000余年的历史。明代时逐渐形成自己的技法和风格，进而取得了很大发展，成为中国著名的一种外销瓷，并在以后数百年间名扬海外，深深影响了“海上丝绸之路”。德化白瓷造型多样，既有实用性的日用器皿，也有观赏性的艺术品，其中以人物雕塑最为著名。它以当地优质的高岭土为原料，制作工艺十分考究，包括陶坯制作、釉料配制、窑炉烧制等多个环节，每一个环节都需要极高的技术水平和丰富的经验积累。

1996年，国务院发展研究中心将德化命名为“中国陶瓷之乡”。2015年，联合国教科文组织世界手工艺理事会授予德化县“世界陶瓷之都”的称号。2024年3月，“中国白 •·德化瓷”国际巡展在美国纽约联合国总部成功举办。

采用多智能体的科普体验
Exploration Experience Featuring Multi-Agent

智能体 (agent) 是能够感知环境、做出决策并采取行动以实现特定目标的自主软件或硬件实体，是大语言模型 (LLM) 应用的高级形态。多智能体德化瓷科普体验采用“谁是卧底”游戏的规则，由基于LLM的多个智能体扮演玩家，通过云端模型调用、提示词组装、上下文记忆等工程手段，驱动多智能体围绕“德化白瓷”主题的秘密词描述特征、思考投票，最终找到“卧底”。

本展示融合了亚马逊云科技在生成式AI技术方面的技术能力与中央美术学院在中国传统艺术领域的专业知识，基于Amazon Bedrock上的Claude3模型，为传播中国传统文化探索新的科普形式，也是双方在生成式AI领域继去年的“皮影随形”展示后又一合作成果。在游戏过程中，您可以引导智能体的思考方向，观察它们的思考行为，从而了解到德化白瓷的相关基础知识，以及生成式AI解决复杂任务的思维过程。

游戏规则
Game Rules

  • 游戏一共有6名玩家，均由基于大语言模型 (LLM) 的智能体 (agent) 扮演。
  • 玩家分为两组，1名是“卧底”，其余5名是“普通玩家”，玩家之间都不知道彼此身份。
  • 游戏给出两个意思相近的词，与瓷器主题相关。普通玩家分到1个，卧底分到另1个。
  • 游戏开始后逐轮进行，每轮分为描述和投票两个环节。每个玩家需要用一句话描述自己得到的主题词，但不能说出词本身；之后，大家投票选出一个出局的对象，最终找出谁是“卧底”。
  • 各智能体根据预设的游戏规则和游戏策略，基于大模型独立思考、观察其他智能体的描述，并决定自己的行动。在第一轮对话中，您可以为其中一个智能体选择思考的角度，引导它的思考行为。

释放云端生图能力
Image Generation on the Cloud with Extension for Stable Diffusion

Stable Diffusion插件解决方案 (Extension for Stable Diffusion on Amazon Web Services) 帮助客户将基于Stable Diffusion web UI的模型训练、推理和调优等任务负载从本地服务器迁移至Amazon SageMaker，利用云上弹性资源加速生图工作，避免单机部署的性能瓶颈。该方案支持txt2img、img2img、ControlNet、ComfyUI、基于Kohya的LoRa模型训练、面向企业应用的多用户管理等功能，并提供web UI和API调用两种使用方式，满足客户多样化的使用场景需求。

轻松切换到Amazon Bedrock上的模型
Easy LLMs Migration with Bedrock Access Gateway

如果您已经在使用OpenAI的API或SDK开发生成式AI应用，想快速切换到Amazon Bedrock但不想修改代码，或者您想使用一些不直接支持Amazon Bedrock的开源工具，但又不想修改其代码，那么可以使用Bedrock Access Gateway解决方案。

它采用无服务器架构，支持一键部署，提供了兼容OpenAI API的代理功能，使用户不需要修改任何原应用的代码，就可以把OpenAI API的调用转发到Amazon Bedrock，从而快速体验包括Claude3系列模型在内的多种LLM的优秀能力。

提示词自动转化与优化
Automatic Prompt Transformation and Optimization

亚马逊云科技研发的开源工具Claude Prompt Generator利用大语言模型，并根据Anthropic公司关于提示词的最佳实践，帮助客户快速将面向GPT的提示词转换和改进为适用于Claude3模型的高质量提示词，并提供效果优化功能，进一步优化提示词在客户场景中的效果，从而协助客户快速实现生成式AI应用针对Claude3模型的适配。该方案提供了一套Python代码和Web应用界面，便于使用者直接操作，或针对批量操作定制。

高效采集、处理与分析业务数据
Efficient Collection, Processing, and Analysis of Business Data

日志通解决方案提供一站式的日志管理与分析功能，基于Amazon OpenSearch等托管服务构建，支持摄取十余种亚马逊云科技服务日志和应用程序日志，能有效地提高日志采集、处理、可视化等场景的工作效率。它提供网页控制台，您不需要编写代码即可轻松创建日志摄取与处理管道，并通过可视化仪表板分析日志数据。

点击流分析解决方案帮助客户轻松收集、处理、分析和可视化来自网页和App的点击流数据，针对特定用户行为或事件快速构建起数据摄取与分析管道，从而精准地开展用户行为分析、产品分析、营销分析，创造业务价值。它提供多平台的前端埋点SDK，图形化的数据管道配置界面，以及开箱即用的可视化报表，并确保客户对埋点数据有完全的控制权。

敏感数据保护解决方案

敏感数据保护解决方案是为企业客户提供的一个亚马逊云科技解决方案，旨在帮助客户自动化识别敏感数据并在统一平台上管理数据资产。该解决方案提供了一个网页应用程序，可跨多个亚马逊云科技账户发现和管理敏感数据，为客户实现数据安全和合规提供技术支持。

Cloudfront预热工具

该解决方案提供用户友好型API，可简化在所选Cloudfront边缘位置预热内容的工作。客户只需调用一次API，即可在数百个边缘位置预热数千个文件。此外，该解决方案还提供清晰的预热任务跟踪，让您轻松监控进度，并在任务完成后查看综合报告。

行业展区

旅游行业智能数据分析助手
Guidance for Hospitality Generative BI Assistant

在旅游行业的数据与业务智能分析的场景中，为了分析业务和经营数据，分析人员经常需要编写多轮、复杂的查询语句以获得业务洞察，同时对于报表外的数据难以进行有效分析。某客户针对该场景，与亚马逊云科技基于零售行业智能数据分析助手解决方案指南一起构建旅游行业智能数据分析助手。借助大语言模型强大的自然语言处理能力，非技术人员也可以通过自然语言对零售环节采购、销售等数据进行查询和分析，不需要掌握SQL等专业技能，帮助业务人员获取数据洞察、提升决策效率。该指南基于Amazon Bedrock、Amazon OpenSearch、Amazon DynamoDB等服务构建。

基于生成式AI的游戏运营分析解决方案指南
Guidance for Game Operation Analysis with Generative AI

在游戏数据分析场景下，游戏运营人员，开发策划人员需要从历史用户事件信息中获得洞察，复杂的表结构和字段含义定义，导致业务团队只能依赖对表和字段结构熟悉的数据团队编写SQL 才能实现个性化的统计分析查询，效率低下。

亚马逊云科技面对该场景打造基于生成式AI的游戏运营分析解决方案指南，用户可以直接使用自然语言提出数据分析任务请求，指南将理解用户意图，并生成合适的SQL查询语句，辅助用户进行数据探索和个性化指标查询。该指南基于Amazon Bedrock、Amazon Redshift、Amazon QuickSight等服务构建。

生成式AI助力工业设计解决方案指南
Guidance for Generative AI Powered Industrial Digital Design

在制造业的工程设计场景下，设计师面临资源受限、团队协作不够流畅以及创意设计过程中高度依赖员工而导致的项目延期等问题。亚马逊云科技构建了生成式AI助力工程设计解决方案指南，使工程设计人员和科学家能够在云上运行传统工作负载，避免了单机性能不足和文件泄露的风险，实现了高效、安全、可控的云上工程设计环境，更可借助Stable Diffusion和Comfy UI构建AI生图工作流。此外，还可以与企业的用户身份系统或管理工作流结合，自动化设计环境申请和软件分发，减少IT运维工作量，优化运营质量。该指南基于Amazon SageMaker、Amazon EC2、Amazon FSx等服务构建。
