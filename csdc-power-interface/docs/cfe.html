业务介绍

背景

随着云计算等技术的快速发展，CDN (内容分发网络) 行业发展加速，市场规模不断提升。Gartner 的研究显示，到2025年，在“传统的”集中式数据中心或云存储以外处理的企业生成数据的比例，将从2018年的10%跃升至75%。Amaon CloudFront是亚马逊云科技的CDN服务，也是客户最常使用的服务之一。当客户有大量的网站使用CloudFront分发时，会产生繁重的日常运维工作。

适用客户

  • 通用行业客户
  • CDN用户

适用场景

客户期望能够围绕Amazon CloudFront提高运维效率、拓展业务功能，主要包括以下应用场景。

  • 迁移：当客户从阿里云CDN、CloudFlare等迁移CDN至亚马逊云科技时，需要批量生成关联SSL证书的CloudFront Distribution。例如，独立站SaaS厂商的站点批量迁移。
  • CDN预热：包括文件预热 (如应用商城)、游戏预热 (各种内容包、升级包) 等。
  • CDN监控：客户需要对下载速度、缓存命中率、带宽等指标进行实时或非实时监控，并且能够通过API集成到其自己统一的CDN监控平台中。
  • CDN版本管理：当客户账号有大量CloudFront Distribution时，需要为某版本的CloudFront Distribution的配置生成快照或查询修改历史。
  • 通用Lambda@Edge函数：客户通过Lambda@Edge函数扩展功能，根据需要一键部署多种等功能，包括图片缩放、鉴权、根据地理位置重定向等。

客户痛点

使用CloudFront的客户通常会面临两方面的挑战：

  • 当客户有大量的网站使用CloudFront时，会产生繁重的日常运维工作。
  • 当客户需要拓展CloudFront的功能时，需要自己开发，学习成本较高。

因此，客户希望有一个统一的平台，在一定程度上实现运维的自动化，并且获得丰富的扩展功能。同时，这些功能可以作为亚马逊云科技的最佳实践供客户参考复用和二次开发。

客户价值

该方案提供了一站式的 Amazon CloudFront 应用与管理的最佳实践，用户可通过一键部署快速实现如下用户场景。

  • 身份验证，例如：通过 Amazon Cognito 或第三方进行身份验证，将安全信息添加到 HTTP 标头中等。
  • URL重写，例如：自动添加 URL 默认路径。
  • URL重定向，例如：根据设备类型返回相应的内容，根据地理位置访问相应的链接。
  • 重写请求，例如：标准化请求格式。
  • 重写响应，例如：修改返回状态码、修改返回 HTTP 标头。
  • 源站选择，例如：根据配置的权重访问相应的源站，源站访问失败后自动跳转到其他源站。
  • 个性化内容，例如：调整图片大小、自定义错误页面。
  • 安全，例如：防盗链等。
  • 其他，例如：预热等。

技术介绍

概要

CloudFront Extensions是一个开箱即用的解决方案，帮助客户高效地运维其亚马逊云科技账号中上百个CloudFront分配 (CloudFront Distribution)。客户可以灵活地按需使用方案多项功能，例如监控、预热、防盗链、版本控制、网站鉴权、根据用户设备类型跳转等。方案当前的v2.0版本在v1.x版本的基础上全面整合了各项功能，客户可以使用方案自带的用户界面或API搭配CloudFront服务使用，轻松管理网站内容的分发。

特点

该方案是一站式的 Amazon CloudFront 最佳实践，可轻松一键部署，同时是开源的，支持用户二次开发。

主要服务

  • Amazon CloudFront
  • Amazon Lambda
  • Amazon Serverless Application Repository (SAR)

部署前提条件

无部署前提条件。客户没有任何基础，也能快速搭建出演示或 POC 系统进行展示和验证。

客户资源/能力评估

可通过 SAR 或 Amazon CloudFormation 一键部署，部署时间大约为30分钟。

成本因素

该方案的成本主要为 Lambda@Edge 和 CloudFront Functions 的使用费用。前者与 Lambda 函数的请求次数、单次运行时间、分配的内存量等因素有关，后者与请求次数有关。

交付资源

该方案提供源代码和Amazon CloudFormation模板。客户可以参考实施指南，运行Amazon CloudFormation模板一键部署。客户可以自行修改源代码来满足特定的需求。如需要咨询技术问题，请在GitHub上提交issue，或通过客户经理联系我们。

更新说明

v2.2

新版本主要包括以下创新和功能优化：

  • CDN预热API全新升级。
    • 支持大批量文件批量预热（如20000个URL）和多达240个Pop点的预热任务下达。
    • 支持按区域、国家、PoP点预热。
  • 新增限速功能。
    • Amazon WAF的速率规则的最低速率限制为100查询/5min。
    • 一些客户的服务URL调用了大量的后端数据库资源和付费的第三方API，因此希望能够使用较低的速率规则来保护某些关键URL。

v2.0

新版本发布，提供更多易用特性。

  • 能帮助客户高效地运维其账号中上百个CloudFront分配，满足CDN迁移 (批量生成关联SSL证书)、预热、监控、版本管理等需求。
  • 支持通过Lambda@Edge函数扩展更多功能，包括图片缩放、鉴权、根据地理位置重定向等。
  • 本方案支持一键部署，开箱即用。
  • 客户可直接使用内置的用户界面，或利用API二次开发。
